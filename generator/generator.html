Fallback content

It's a good practice to include fallback content that will display if the SWF cannot be embedded. Two common approaches are:

  1. Display a message informing the user they need to install/upgrade Flash Player, along with a link to Adobe's Flash player installer.
  2. Display an HTML representation of the content contained in the SWF. This ensures visitors will have access to your site's information even if they don't have Flash Player.

Advanced Options

Optional Attributes for the <object>
DYNAMIC PUBLISHING ONLY. By default, SWFObject's dynamic publishing will give the new object the same ID as the HTML element that is being replaced (listed above under "ID you'd like to use for your SWF"). Only specify an ID here if you want the generated <object> to have a different ID than the original HTML element. In most cases you should leave this field blank.
Adds a "name" attribute to the <object>. The name attribute is not usually needed. In most cases you should leave this field blank.
Adds a "class" attribute to the <object>. This is usually used for styling via CSS or for advanced JavaScript functionality.
HTML alignment of the object element. If no alignment is specified, the SWF will default to "left". In most cases you should leave set to "none".

FlashVars

Enter the FlashVars using the format name=value. Do not use quotation marks. Enter only one FlashVars per line.

Example:

artist=Pale Saints
cd=In Ribbons
year=1992
